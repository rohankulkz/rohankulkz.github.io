Using AI to Process over 30,000 Hours of User data

Providing high quality asynchrounous data analytics effeciently and at scale

Tech Stack

  • AWS DynamoDB
  • AWS Lambda
  • AWS Comprehend
  • GPT-4.1-Nano
  • AWS SQS

System Design

I needed fast and effecient data analytics on rapid conversation data, generated live and with conversations. Data was being processed and stored via a serverless Lambda function which placed data in DynamoDB. To integrate data analytics into the preixisting system, I utilzied an async queue which communicated data to an analytics function. This provided the advantages of an async system while also allowing for rapid analysis and updates. The analytics were a combination of AWS Comprehend requests to extract sentiment and conversation themes as well as LLM use to extract meaning and thematic shifts through conversation. This data was valuable to customers and was used for effective product placement, targeted advertisement, and general market analysis
